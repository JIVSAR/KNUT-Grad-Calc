## 1. 전면 디자인 개편 (Tailwind 기본 → "파스텔 일러스트" 디자인 시스템)

**배경**: 초기 UI가 Tailwind 기본값(흰 카드·기본 블루·이모지)이라 "스타터 템플릿" 느낌. 사용자 피드백: "개쩐다 싶은 게 없다."

**과정 (디자이너식 시안 제시)**:
1. 병렬 워크플로(subagent fan-out)로 디자인 방향 5종 → 사용자 평가 "평범".
2. 2차: 소프트 파스텔 기반 4종(라이트/다크 토글 포함) → **H "파스텔 일러스트"** 채택.
3. 와우 3요소 확정: **색감/그라디언트 · 움직임/애니메이션 · 공간감/깊이**.

**구현**:
- `src/index.css`: 라이트/다크 CSS 토큰(파스텔 p1 보라 #7C6FF0 · p2 핑크 · p3 민트 · p4 앰버), 오로라 그라디언트 배경 + 블러 블롭, 글래스 카드(backdrop-filter), 그라디언트 진행링/바, 공유 컴포넌트 클래스 카탈로그(field·btn·segmented·group-box·list-row·modal·banner·data-table·upload-box 등).
- Pretendard 폰트(index.html CDN).
- **테마 시스템**(`src/state/theme.ts`): 라이트/다크/시스템, localStorage `knut-theme`, FOUC 방지 인라인 스크립트. 헤더 토글 + 설정(백업) 탭 3종 선택.
- 애니메이션: 진행링 채움(stroke-dashoffset), 숫자 카운트업(`lib/useCountUp`), 카드 순차 등장(`.reveal`/`.stack`), 탭 전환.
- 신규 컴포넌트: `Ring` · `ThemeToggle` · `ThemeSetting` · `Modal`(포털) · `UploadBox`.
- 대시보드 전면 재구성 후, 나머지 4탭은 **병렬 워크플로**로 잠근 클래스 카탈로그를 적용(로직 보존, className만 교체).

**테마 전환 일관성 (트러블슈팅)**:
- 증상: 다크↔라이트 전환 시 카드 배경만 서서히 변하고 헤더·그라디언트·텍스트는 즉시 바뀜(반쪽짜리).
- 원인: 일부 요소에만 `transition: background .5s`. 게다가 그라디언트는 CSS `transition` 대상이 아님(즉시 변경).
- 해결: 반쪽 CSS 전환 제거 + 테마 적용을 **View Transitions API**(`document.startViewTransition`)로 감싸 화면 전체를 한 번에 크로스페이드. 미지원·모션축소 시 즉시 전환.
- 검증: `startViewTransition` 호출 확인, 토큰 전환 확인.

## 4. 트러블슈팅 로그 (증상 → 원인 → 해결 → 검증)

### 4.1 모든 모달이 화면 하단에 잘려서 뜸
- 증상: 삭제 확인·성적 입력 등 모달이 화면 중앙이 아니라 콘텐츠 하단에 잘려 표시(전체 초기화 포함 모든 모달).
- 원인: `position:fixed` 모달이 조상 요소(애니메이션 transform 등)의 컨테이닝 블록에 갇혀 뷰포트 전체가 아닌 콘텐츠 영역 기준으로 배치됨.
- 해결: 공용 `Modal.tsx`로 통일 — `createPortal`로 `<body>` 직속 렌더 + `.modal-overlay align-items:center`. ConfirmModal·CourseEditModal·CompleteSemesterModal·가드·재수강 모두 포털.
- 검증: 오버레이가 `document.body`의 직계 자식·중앙 정렬 확인.

### 4.2 성적 입력 모달에 과목명이 사라짐
- 증상: 학기 수강완료 모달에서 "미입력 / 미입력"만 보이고 어떤 과목인지 모름.
- 원인: 디자인 적용 중 과목명 span에 `truncate`가 잘못 걸려 잘림 + 모달 하단 잘림(4.1).
- 해결: 행 레이아웃을 `.list-row`(과목명·영역·학점 + 성적 select)로 재작성. 4.1 포털로 위치 정상화.
- 검증: 7~8개 과목 모두 이름+영역·학점+성적칸 표시 확인.

### 4.3 비학점 졸업요건 체크가 전체 진행률에 반영 안 됨
- 증상: 대시보드 히어로 진행률이 학점(총학점/140)만이라 비학점 인증 체크해도 안 변함.
- 원인: 진행률이 `total.earned/total.min`만 사용.
- 해결: 엔진 `overallProgress` 추가 — 학점 영역(total 제외)·GPA·필수과목·비학점 인증의 충족 비율 평균. 히어로 링이 이걸 사용.
- 검증: 비학점 체크 시 37%→46% 변동 확인.

### 4.4 대시보드와 계획 탭의 "계획 반영 시 남은 학점"이 다름
- 증상: 같은 항목인데 대시보드 23, 계획 탭 38로 불일치(둘 다 부정확).
- 원인: '총 졸업학점' 영역이 **선택한 다전공 트랙에 속한 카테고리 학점만** 셈. 트랙 밖 카테고리(예: 전공심화인데 K-융합 과목)가 140 계산에서 빠짐. 계획 탭은 모든 계획학점, 대시보드는 필터된 학점을 표시.
- 해결: **총 졸업학점 140은 카테고리 무관 모든 학점**으로 계산하도록 `evaluate`·`project`의 total 영역 수정.
- 검증: 양쪽 모두 동일(계획 20 → 39) + 산수 일치.

### 4.5 수강내역 재업로드 시 중복 + 삭제 오작동
- 증상: 이수완료 일부 삭제 후 같은 수강내역 재업로드→수강완료 시 이수완료에 **중복**. 그 상태에서 삭제 버튼이 엉뚱하게 동작(화면 안 지워지고 더 생김).
- 원인 ①: 엑셀 파서가 ID를 매 업로드마다 `e0,e1…`로 리셋 생성 → 기존 과목과 **ID 충돌** → React key 충돌로 삭제 오작동.
- 원인 ②: 수강내역 재업로드가 이미 이수완료된 같은 학기 과목을 다시 수강중으로 추가 → 중복.
- 해결: ① import 시 **모든 과목에 새 고유 ID(UUID)** 부여. ② 수강내역 병합 시 **같은 학기 이수완료 과목은 건너뜀**(`mergeImportedEnrollment` 중복 방지). 지운 과목만 수강중으로 복귀.
- 검증: 50과목 ID 중복 0, 삭제 정확히 1개. 단위테스트 추가.

### 4.6 같은 파일 재업로드가 안 됨
- 증상: 전부 삭제 후 같은 수강내역 엑셀 재선택 → 수강중 안 뜸.
- 원인: HTML 파일 입력은 **동일 파일 재선택 시 `onChange` 미발생**(값 불변).
- 해결: `UploadBox` onChange 후 `e.target.value=''`로 초기화 → 같은 파일 재선택 가능.
- 검증: 실제 엑셀 e2e 업로드(8과목 파싱·반영) + 입력 값 초기화 확인.

### 4.7 성적표 재업로드가 수동 완료한 학기를 덮어씀
- 증상: 2-2까지만 있는 성적표 재업로드 시, 수강내역으로 수동 완료한 3-1이 사라짐.
- 원인: `mergeImportedTranscript`가 **기존 이수완료 전체를 교체**.
- 해결: 성적표는 **포함한 학기만 갱신**, 다루지 않는 학기의 이수완료는 보존(학기 있으면 학기 기준, 없으면 코드/이름 기준).
- 검증: 2-2 성적표 재업로드 시 3-1 보존, 2-2는 중복 없이 갱신.

### 4.8 코드 없는 수동 추가 과목이 재수강 처리 안 됨 → 모달로 전환
- 증상: 성적표 자료구조(코드 有)와 같은 이름을 수동 추가(코드 無)해도 자동 대체 안 됨.
- 1차 해결: dedup 키를 union-find로 이름↔코드그룹 통합(코드 유무 무관 같은 이름 묶음).
- 사용자 요청 변경: "자동 대체 말고 모달로 물어봐."
- 최종: dedup 자동 대체는 **코드 일치(성적표 재수강·동일교과목)만**. 수동 추가 시 같은 이름이 있으면 **"재수강 처리할까요?" 모달**(재수강 처리 / 그냥 추가 / 빨간 취소). 확인 시 기존 과목을 retake 표시(`addCourseRetaking`).
- 검증: 모달 동작·재수강 표시·취소 시 미추가 확인.

### 4.9 성적 미입력 강제 + 강조 애니메이션
- 요청: 수강완료(이수 이동) 시 성적 전부 입력 필수, 미입력 시 강조.
- 해결: 미입력 과목 select에 `.field-error`(빨강 + shake 키프레임) + 안내 배너, 미입력 있으면 이동 차단. 재시도마다 애니메이션 재생(잠깐 비웠다 채움). 버튼명 "이수로 옮기기"→"수강 완료".
- 검증: 미입력 8개 강조·차단, 성적 입력 시 강조 즉시 해제(8→7).

### 4.10 수강중 과목이 "반영 시" 계산에 누락
- 증상: 대시보드/계획 탭 "계획 반영 시 남은 학점"이 예정만 반영, 수강중 누락.
- 해결: 엔진에 `totalEnrolled`·영역별 `enrolled` 추가. 대시보드 카드 "수강중·계획 N학점 반영 시 → M"(적응형 라벨). `project()`가 수강중+예정을 함께 반영(표 컬럼에 수강중 학기 등장). 단, 계획 탭 아래 편집 목록은 예정만(수강중은 과목 탭 관리).
- 검증: 수강중6+계획6=12 → "수강중·계획 12 반영 시 29"(대시보드·계획 일치).

### 4.11 이수한 과목의 재수강(수강중)이 남은 학점을 잘못 깎음
- 증상: 이수 자료구조(3학점) + 같은 자료구조 수강중 재수강 → "수강중 3학점 반영 시 → 134"(137이어야 맞음). 재수강은 새 학점이 아님.
- 원인: 반영 계산이 재수강을 새 학점으로 카운트.
- 해결: 반영 계산에서 **이미 이수한 과목(코드 기준)의 재수강 예정/수강중 제외**(`evaluate`·`project`). 낙제(F) 과목 재수강은 이수 못 했으니 새 학점으로 반영.
- 검증: 137→137(안 깎음), 낙제 재수강은 totalEnrolled 3, 단위테스트 2개 추가.

## 6. 2차 작업 세션 (2026-06-24) — 일관성 전수 감사 · UI/UX 정밀화 · 재수강 모델 정립

> 이 세션은 "사용자가 화면을 처음부터 꼼꼼히 재검토하며 발견한 문제들"을 **하나씩 진단→수정→라이브 검증**하는 흐름. 매 수정마다 `tsc` + Vitest + 브라우저 미리보기 e2e(localStorage 시드, DOM 구동/eval, 실제 xlsx 주입)로 검증. 테스트 43→54.

### 6.1 발단 — "총 졸업학점" 기준 수정이 화면에 반영 안 됨 (two sources of truth)
- 증상: 요건 탭에서 총 졸업학점을 100으로 바꿔도 대시보드·플래너는 140으로 그대로. 다른 영역(전공/교양 등)은 반영됨.
- 원인: "총 졸업학점"이 스펙에 **두 곳**에 중복 저장 — 최상위 `totalMinCredits`(요건 탭 input이 수정) vs `areas[id:'total'].minCredits`(화면이 읽음). `applyOverrides`가 영역 쪽을 안 맞춰줌 → 갈라짐.
- 해결: `applyOverrides`에서 'total' 영역의 minCredits를 항상 유효 `totalMinCredits`에 동기화(단일 진실원). 대시보드 카드·히어로·플래너 총합 행이 한 번에 정상화.
- 검증: 라이브 — 히어로 `80/100`, 카드 `20학점 남음`, 플래너 총합 행 `20|2|0`. 회귀 테스트 추가.

### 6.2 데이터 일관성 전수 감사 (Workflow · 8차원 · 32 에이전트) → 15건 수정
- 동기: 6.1이 "한 개념이 여러 곳에 저장돼 어긋나는" 유형. 같은 클래스가 더 있을지 의심 → 전수 감사.
- 방법: **Workflow 도구**로 8개 차원(① 중복진실/쓰는곳≠읽는곳 ② override 전파 ③ composeSpec/다전공 ④ evaluate↔project 분기 ⑤ 스펙 데이터 정합 ⑥ 영속/백업 라운드트립 ⑦ 죽은 코드 ⑧ 학점 집계) 병렬 finder + **각 발견을 적대적으로 검증**(진짜 버그 vs 의도/오탐). 32 에이전트, 24발견 → **15확정 / 9기각**.
- 두 뿌리 원인: **(A) 학점 기준값의 단일 진실원 부재**(composeSpec의 majorTotalOverride · applyOverrides의 areaMin · 원본이 같은 area.minCredits를 덮어써 누가 이기는지 불명확), **(B) Course 상태 플래그를 수동 화이트리스트로 복원**(backup.parseBackup이 새 플래그를 누락) → 전면 재작성 불필요, 4개 파일 집중 정합.
- 확정 15건 수정(요약):
  - 🔴 **JSON 백업 복원 시 `enrolled` 누락** → 수강중이 이수로 둔갑(학점·GPA 부풀림). `parseBackup`에 한 줄 추가 + 회귀 테스트(원래 통과시키던 테스트 데이터에 enrolled 과목 없었음).
  - 🟠 **majorTotal(전공계) 트랙 전환 desync**: cs2022에서 전공 합계를 한 번 수정하면 `areaMin.majorTotal`이 박혀, 다전공 트랙을 바꿔도 `majorTotalOverride`(90↔69)가 무시됨. `setMultiMajor`에서 트랙 종속 override 정리.
  - 🟠 **트랙/학번 전환 시 과목 고아화**: 새 트랙에 없는 categoryId 과목이 영역 합계에선 빠지나 총학점엔 남아 어긋남. `evaluate`에 `unclassified` 집계 + 대시보드 경고 카드.
  - 🟡 예정·수강중 동일교과목 **중복 미제거**, `remainingAfterPlan`이 enrolled 미반영, `areaMin['total']` 잠복 desync, per-area 되돌리기 버튼.
  - ⚪ 죽은 코드(`NonCreditType`·`RequiredCourseResult.categoryId` 제거, `currentCurriculumYear`는 대시보드에 표시해 살림).
  - **import 미지원 이수구분('부전공'/'복수전공')**: 자동 분류 못 함. *추측 금지* — 표기를 모르니 하드코딩 대신 **"모르는 이수구분을 원본 표기 그대로 보존 + 경고로 표면화"**(성적표=조용히 버리던 것/수강내역=전선 둔갑 → 둘 다 해결). 실제 표기는 사용자 확인 시 1줄 매핑으로 마무리.

### 6.3 커스텀 드롭다운 컴포넌트 (네이티브 `<select>` 전면 교체)
- 증상: 온보딩 등 셀렉트의 펼친 목록이 OS 네이티브라 글래스 다크 테마와 안 맞음(회색 시스템 팝업).
- 해결: 신규 `Select.tsx` — **포털(document.body)** 로 메뉴를 빼 카드의 `backdrop-filter` 쌓임맥락에 안 가리게, 글래스 테마·열림 애니메이션·외부클릭/Escape·`placeholder`(트리거엔 회색 안내문구, 목록엔 미표시)·`error`(빨강 강조) prop. 앱 전 셀렉트(영역·성적·학년·학기·학과학번·다전공·학점) 교체. 공유 `SemesterSelect`/`AlsoCountsSelect`도 내부 전환.
- 트러블슈팅: **드롭다운 스크롤 안 됨** — "스크롤 시 닫기" 핸들러가 메뉴 내부 스크롤까지 닫기로 인식. `e.target`이 메뉴 내부면 닫지 않도록 + `overscroll-behavior: contain`.

### 6.4 그룹 펼치기/접기 애니메이션 — **삼중 트러블슈팅** (브라우저/헤드리스 한계)
- 목표: 과목 탭 학기 그룹 접기를 즉시 토글 → 부드러운 높이 전환.
- 1차(grid): `grid-template-rows: 0fr↔1fr` + transition. **런타임 접기가 동작 안 함**(정적 초기 상태만 0px, 클릭 토글은 0.4초 뒤에도 높이 유지). 이 브라우저에서 fr 트랜지션이 런타임에 미작동.
- 2차(rAF): 콘텐츠 높이 측정해 `height` 트랜지션. **헤드리스 프리뷰에서 `requestAnimationFrame`이 스로틀**돼 `setHeight(0)` 콜백 미발생 → 검증에서 또 실패.
- 3차(최종, `Collapse.tsx`): rAF 제거. **직접 DOM 조작 + 동기 리플로우(`void el.offsetHeight`)로 시작 높이 등록 → 0으로 트랜지션**, 펼침은 `scrollHeight`→`transitionend`에 `auto`. `prefers-reduced-motion` 존중. 캐럿은 transform 회전(transform은 잘 됨). 라이브로 접힘 0px/펼침 전체높이 확정.

### 6.5 과목 추가 폼 — 필수 입력 검증 + placeholder 기본값
- 기본 자동선택 제거: 학점(3)·영역(전공필수)·성적(미입력)이 흰 글자로 자동선택돼 있던 것 → **학년·학기처럼 회색 placeholder**("학점"/"이수 영역"/"미입력"), 아무것도 안 고른 상태로 시작.
- 필수 검증(수강 완료 성적 강조 패턴 재사용): **이수 완료=과목명·학점·영역·학년·학기·성적(6)**, **수강 계획/수강중=성적 제외(5)**. 빈 채로 추가 시 빈 필드 `.field-error`(빨강+shake) + "필수 항목을 입력해 주세요" + 추가 차단. 입력하면 강조 즉시 해제. 라이브 E2E(빈→6강조→채움→추가 성공→폼 초기화).
- 학점은 자유입력 → **드롭다운(1·2·3·4·5·6·9)**, 기존 비표준 학점값(예: 가져온 7학점)은 옵션에 보존.

### 6.6 재수강 모델 정립 — **설계 진화** (핵심 트러블슈팅)
- 증상: 2학년에 이수한 자료구조를 계획 탭에서 3-1 재수강 계획으로 넣으면 **학기별 남은 학점이 깎임**(재수강은 새 학점이 아닌데).
- 원인: dedup/projection의 재수강 인식이 **교과목코드 기준** → 수동 추가(코드 없음) 과목은 이름이 같아도 안 묶임(`getSuperseded`도 코드 없으면 패스).
- 1차: projection/evaluate를 **이름+코드 자동 인식**으로 확장(코드 없어도 이름 같으면 재수강 → 새 학점 제외). 낙제(F) 과목 재수강은 이수 못 했으니 새 학점 인정. 학기 열은 보이되 학점 변동 0.
- 사용자 지적: "예전 세션에서 과목 탭처럼 **재수강 확인 모달**을 만들었었는데?" → 명시적 확인 선호(과목 탭은 모달 有, 계획 탭은 없던 사각지대).
- 2차: 계획 탭에 **재수강 확인 모달**(재수강으로 추가 / 그냥 추가 / 취소) 추가, 자동인식→명시적 플래그로 전환. *문제*: 새 과목에 기존 `retake` 플래그를 썼더니 그게 **"재수강으로 대체된 이전 과목"(취소선·dim) 의미**라 새 과목이 취소선 처리됨.
- 3차(의미 분리): 신규 플래그 **`retaking`**(재수강 중/예정 — "재수강" 배지, 학점 미반영) vs 기존 **`retake`**(대체된 옛 과목 — 취소선). 옛 과목은 **계획 단계에선 그대로 유효**(아직 재수강 안 끝났으니 학점 살아있음 → projection 불변).
- 4차(완료 시 대체): 재수강을 **'수강 완료'하면** 그때 옛 과목이 `retake`(대체됨)로 바뀌고 새 과목이 새 성적으로 정식 이수 → **이중계산 방지**(`completeCourses` 수정). `retaking` 플래그는 backup 보존.
- 검증: 라이브 E2E(계획 입력→모달→재수강으로 추가→배지·취소선X·projection 불변→수강 완료→옛=대체됨/새=A0 이수·중복0) + 단위테스트(retaking 미반영 / 그냥추가 반영 / 동시인정 / 코드 자동).

### 6.7 기타 UI/UX 정밀화 (라이브 측정으로 검증)

### 6.8 검증 방법론 (이 세션에서 확립)
- 매 수정: `tsc --noEmit`(로컬 `node_modules/.bin/tsc` — `npx tsc`는 엉뚱한 패키지 받음) + `npm test`(Vitest 54) + **라이브 프리뷰 e2e**(localStorage에 시나리오 시드 → 리로드 → DOM 구동(클릭/입력/eval) → 상태·렌더 측정 → 검증 후 시드 정리).
- **헤드리스 프리뷰 한계 발견·우회**: rAF 스로틀(6.4), CSS 트랜지션 즉시처리로 중간 프레임 미캡처(끝/시작 상태로 검증), 스크린샷 타임아웃(eval 측정으로 대체), React state 비동기(같은 eval에서 직후 측정 금지 → 별도 eval).
- **추측 금지 원칙**: 성적표 이수구분 표기(부전공/복수)를 모를 때 하드코딩 거부 → "모르는 표기 자동 표면화 + 사용자 확인" 설계.

## 7. 3차 작업 세션 (2026-06-25) — 재수강 GPA 정합 · UX 애니메이션 · 반응형 · **배포**

> 흐름: 재수강 성적 규칙 버그 수정 → UX 다듬기(토스트·스크롤바·전환 애니메이션) → **Vercel 배포** → 실폰 반응형 수정. 까다롭고 *눈으로 검증 불가*한 결정(애니메이션·스크롤바)은 **Workflow(정찰→설계→적대적 검증)**로 굳히고, 코드 측정 + 사용자 캡쳐로 검증. 테스트 54→**61**.

### 7.1 재수강 '높은 성적 우선' GPA 버그 (도메인 정합)
- 증상: 2-1 자료구조 **C+** 이수 후 3-1 **C0** 재수강 완료 → GPA가 낮은 C0로 계산(높은 C+여야 함).
- 원인: 코드기반 dedup(`getSuperseded`)은 '최고 성적 유지'로 옳지만, **수동 재수강 경로**(`completeCourses`·`addCourseRetaking`)가 성적 비교 없이 **무조건 최신 과목으로 대체**(옛 과목에 `retake`). `retake`를 박으면 grade-aware 비교에서도 빠져 낮은 성적이 굳음.
- 해결: 공용 **`gradeScore`·`bestGradeId`**(`dedup.ts`) 추출 → 두 경로가 같은 과목 그룹에서 **최고 성적만 유효(retake 아님)**, 나머지 `retake`. (cap: 재수강 자격 C+↓·취득 상한 A0는 사용자 확인됐으나 별도 미구현)
- 검증: 단위테스트 +7=61. 라이브 e2e — 2-1 C+ + 3-1 C0 재수강 완료 → GPA **2.50(C+)**, C0 취소선.

### 7.2 토스트 알림 (피드백 위치)
- 증상: 백업 탭 상단 "파일로 저장" 눌러도 확인 메시지가 한참 아래(복원 카드 뒤)에 떠 안 보임. 원인: 단일 `.banner`가 JSX 흐름에 고정 → 분산된 트리거와 어긋남.
- 해결: `Toast.tsx`(body 포털·상단 고정·자동소멸 ok 2.5s/err 6s·`flushSync`로 상태 동기 커밋·**persistent live region**으로 스크린리더 보장). Workflow 적대적 검증이 라이브영역 상시 마운트·중앙정렬 단일화·에러 더 길게를 잡음.

### 7.3 슬림 스크롤바 (공용 `.slim-scroll`)
- 드롭다운·textarea·모달·페이지(html)의 OS 회색 스크롤바를 글래스 테마 8px 둥근 thumb로 교체. **함정**: 표준 `scrollbar-width`를 함께 쓰면 Chromium이 `::-webkit-scrollbar` 커스텀을 무시 → `@supports not selector(::-webkit-scrollbar)`로 Firefox 폴백만 분리. 둥근 모서리 밖 삐짐은 트랙 위·아래 인셋으로.

### 7.4 상태전환 애니메이션 — 모핑 폐기, 도착 효과 채택 (설계 진화)
- 목표: 학기 시작(planned→enrolled)·수강 완료(enrolled→completed)를 부드럽게.
- 1차(View Transitions 모핑): 행에 `view-transition-name` → 행이 날아감. **그러나** 반투명 글래스+내용 변화(배지↔성적)로 옛/새 스냅샷이 크로스페이드되어 글자가 **"분신술"(고스팅)**. 게다가 두 구역이 모두 `overflow:hidden`이라 가로지르는 비행은 잘림.
- 적대적 검증 결론: 깔끔한 비행은 body 포털 클론이 필요(과함). → **모핑 폐기**.
- 최종: `markArrivals`(flushSync 후 새 위치 행에 `.row-arrive` 슬라이드+하이라이트). 스냅샷이 없어 고스팅·잘림 원천 차단. (테마용 `viewTransition.ts` 헬퍼는 만들었다 제거 — `theme.ts`가 자체 `startViewTransition` 사용.)

### 7.5 반응형 (실폰 검증)
- **한글 줄바꿈**: 좁은 폰에서 "졸업학점"→"졸업학/점"처럼 단어 중간 끊김 → `body`에 `word-break:keep-all`+`overflow-wrap:break-word`(전 텍스트 상속).
- **헤더 문구**: eyebrow "졸업까지 함께"→"국립한국교통대학교", 제목 "국립한국교통대학교 졸업학점 계산기"→"졸업학점 계산기".
- **폼 셀렉트 겹침**: 과목 추가/수정의 학기·성적 셀렉트가 좁은 화면에서 겹침. 원인: flex/grid 자식 기본 `min-width:auto`라 내용보다 못 줄어 칸 넘침 — 그리고 **flex 아이템이 트리거가 아니라 `.select` 래퍼**임을 rect 측정으로 발견. → `.select`·`.field-label`·`.select-trigger`에 `min-width:0`. 이어 칸 비율 조정(학기↑·성적↓ `minmax(0,1fr) minmax(0,0.42fr)`)+`.tight-selects`(패딩/캐럿 축소)로 '여름학기'(4글자) 수용. **검증**(`preview_resize`+rect/scrollWidth 측정): 360px+ 겹침·잘림 0(폼·모달), 320px만 4글자 학기·placeholder 살짝 잘림.
