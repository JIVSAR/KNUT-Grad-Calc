# 한국교통대 졸업학점 계산기 (KNUT Graduation Calculator)

한국교통대학교 학생이 **졸업 진행 상황**과 **다음 학기 수강 계획**을 휴대폰에서 한눈에 확인하는 PWA.
1차 대상: 충주캠퍼스 **컴퓨터공학과 2024학번** (요건을 데이터로 분리해 전 학과 확장 가능 구조).

**🔗 라이브 데모: https://knut-grad-calc.vercel.app** — 폰으로 접속 후 "홈 화면에 추가"하면 앱처럼 쓸 수 있어요.

> 만든 이유: 매 학기 "졸업까지 몇 학점/어떤 과목을 더 들어야 하나"를 손으로 계산하기 귀찮아서.
> 성적표만 올리면 자동 집계되고, 남은 학기 계획을 짜면 학기별로 남는 학점이 자동 계산된다.

## 주요 기능
- **졸업 판정**: 총학점·전공·교양·다전공·GPA·이수학기·전공필수 과목·비학점 인증을 종합해 졸업 가능 여부와 미충족 항목 표시
- **성적표 엑셀 업로드**: 통합정보시스템 성적 엑셀(`개인성적.xlsx`)을 올리면 이수 과목 자동 추가(이수구분·학점·등급·학기 파싱). 재업로드 시 **똑똑한 병합**(수동 설정 보존 + 성적 나온 학기의 계획 자동 정리)
- **학기별 수강 계획**: 들을 예정 과목을 학기별로 추가하면 **그 학기까지 들었을 때 남는 학점**을 영역별로 자동 계산 (수기 계획표를 자동화)
- **재수강/동일교과목 처리**: 같은 코드 또는 동일교과목이면 **최고 학점만** 인정(학점 1회), F/NP 미취득. 학칙 cap 반영 — 재수강 자격은 원 성적 **C+ 이하**, 취득 성적 **상한 A0**
- **다전공 선택제**: K-융합전공·부전공·복수/연계·소단위·융합전공 중 선택 → 해당 요건 적용
- **과목 동시 인정**: 전공이면서 K-융합 등 한 과목이 여러 영역에 동시 인정(총학점엔 1회)
- **GPA**(KNUT 4.5 만점) · 비학점 인증(영어/비교과/졸업시험) 체크리스트
- **다학번 지원**: 입학년도별 요건(24·22학번 등) + 재학 중 최신 편성표의 전공필수 공유 모델
- **과목 상태 흐름**: 이수 완료 / 수강 중 / 들을 예정 3단계 + 학기 시작·수강 완료 흐름, 수강내역 엑셀 업로드
- **라이트/다크 테마**(시스템 연동, View Transitions 크로스페이드) · 좁은 폰 대응 반응형
- 로컬 저장 + JSON 백업/복원, 설치형 PWA(오프라인)

## 기술 스택
Vite · React 19 · TypeScript · Tailwind CSS v4 · vite-plugin-pwa · Zustand(persist) · Vitest · SheetJS

## 설계 포인트
- **요건 = 데이터, 로직 = 공용**: 학과별 졸업요건은 `src/data/requirements/cs2024.ts` 한 곳에. 엔진/UI는 스펙을 받아 동작 → 학과 추가가 *데이터 파일 추가*로 끝남.
- **순수 함수 엔진 + 단위 테스트**: 집계·판정·GPA·projection·중복처리를 순수 함수로 분리해 Vitest로 검증(66개).
- **다전공/동시인정 합성**: 기본 요건 + 선택한 다전공을 `composeSpec`으로 합치고, 영역 합계는 과목당 1회로 계산(중복 인정해도 총학점은 1회).

## 개발 과정 (요약)
1. **요건 데이터 확보** — 학교가 공개한 자료(졸업안내·졸업자격인증제 운영지침·학사운영규정 별표3·교육과정편람)와 통합정보시스템 졸업자가진단 화면에서 수치를 **출처와 함께 확정**. *추측 없이, 못 찾은 값은 "확인 필요"로 표시 후 사용자 캡쳐로 확정.*
2. **자료 추출** — 학교 문서가 HWP/이미지/xlsx라, `olefile` + 자체 HWP 레코드 파서, `openpyxl`, SheetJS로 텍스트/표를 추출해 정확히 반영.
3. **반복 개발** — 졸업판정 → 성적표 업로드 → 학기 계획 → 재수강 처리 → 다전공 선택제 → 동시인정 → UX(그룹·토글·모달) 순으로 사용자 피드백을 받아 기능 추가.
4. **검증** — 매 단계 Vitest + 브라우저 미리보기 e2e(실데이터 주입)로 수치·동작 확인.

### 해결한 기술적 과제
- HWP(한글 OLE) 본문을 라이브러리 없이 zlib+레코드 파싱으로 추출
- "K-융합전공"과 "융합전공"이 별개 제도임을 학칙에서 확인해 분리
- 재수강·동일교과목 중복을 교과목코드 + 동일교과목 목록(union-find)으로 묶어 최고학점만 인정
- 한 과목의 다영역 동시 인정(전공↔K-융합)을 총학점 1회 보장하며 구현
- 성적표 재업로드 시 사용자 수동설정 보존 + 학기 기준 계획 정리

## 실행
> 이 PC는 PowerShell에서 npm이 막혀 있어 **Git Bash**로 실행.
```bash
npm install
npm run dev      # http://localhost:5173
npm test         # 단위 테스트
npm run build    # 프로덕션 빌드 → dist/
```

## 배포
**🚀 라이브: https://knut-grad-calc.vercel.app** — Vercel(Vite 자동 감지, build `npm run build` / output `dist`). `main`에 push하면 자동 재배포.
